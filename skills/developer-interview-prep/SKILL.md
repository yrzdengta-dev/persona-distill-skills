# Developer Interview Prep

## Purpose

帮助软件开发岗位候选人准备技术面试中的关键环节，包括简历、项目叙事、技术问答、系统设计和面后复盘。

核心目标是回答这些问题：

1. 开发岗简历是否能支撑技术追问？
2. 项目经历是否讲得出真实 ownership、技术取舍和结果？
3. 面试官最可能从哪些角度深挖？
4. 哪些短板最可能在技术面或系统设计环节暴露？

## When To Use

- 需要准备后端、前端、全栈、基础设施、测试开发等开发岗位面试
- 需要诊断开发岗简历
- 需要打磨项目介绍和自我介绍
- 需要模拟技术面或系统设计面
- 需要复盘一次真实开发岗面试

## Inputs

- 简历正文或相关片段
- 目标岗位 JD
- 项目经历、技术栈、作品或仓库链接
- 已经被问过的技术题或系统设计题
- 用户想重点补强的技术环节

## Outputs

- 开发岗简历问题清单与修改建议
- 项目经历深挖提纲
- 技术面高概率问题列表
- 系统设计准备提纲
- 模拟面试追问与点评
- 面后复盘与补强计划

## Workflow

1. 明确岗位与技术范围
- 先确认岗位方向：后端、前端、全栈、客户端、基础设施或测试开发。
- 再确认级别、语言栈、核心职责和面试阶段。

2. 评估简历是否支撑技术追问
- 检查项目描述是否能经得起“你具体做了什么”的追问。
- 重点看：
  - ownership 是否清楚
  - 技术选型是否有因果
  - 性能、稳定性、质量指标是否可信
  - 是否存在技术名词堆砌

3. 识别高概率考点
- 通用考点：
  - 项目深挖
  - 调试与故障定位
  - 性能优化
  - 工程质量
  - 协作与取舍
- 后端附加考点：
  - 并发、数据库、缓存、消息队列、API 设计、可靠性
- 前端附加考点：
  - 状态管理、渲染性能、浏览器机制、工程化、可访问性
- 全栈附加考点：
  - 前后端边界、接口契约、数据一致性、端到端交付

4. 组织技术回答
- 把项目和问题回答整理成统一结构：
  - 背景
  - 问题
  - 约束
  - 方案
  - 取舍
  - 结果
  - 复盘
- 如果回答里缺少取舍和边界，默认还不够成熟。

5. 模拟技术面与系统设计
- `技术面模拟`：围绕简历和技术栈逐层追问
- `系统设计模拟`：围绕需求拆解、数据流、扩展性、风险和取舍追问
- 每轮结束后给出：
  - 技术表达问题
  - 证据不足点
  - 深挖风险点
  - 下一轮补强建议

## Preparation Rules

1. 真实性优先
- 不虚构项目、指标、职级或 ownership。
- 可以优化叙述，但不能篡改事实。

2. 深度优先于术语
- 会背技术名词不等于能过技术面。
- 先把自己做过的东西讲清楚，再谈扩展知识。

3. 取舍优先
- 技术面试最容易区分人的地方，不是知道多少术语，而是能不能讲清为什么这样做。
- 每个关键回答都应尽量体现限制、方案比较和权衡。

4. 风险识别优先
- 主动识别最容易翻车的点：
  - 项目里写了但讲不深
  - 只知道 happy path，不知道故障与边界
  - 只会说“用了什么”，不会说“为什么这样用”
  - 性能优化和工程质量写得太空

## Default Output Modes

### 1. 开发岗简历诊断

输出：

1. 总体匹配度
2. 最容易被技术面打穿的点
3. 项目描述修改建议
4. 应删减和应补强的内容

### 2. 项目深挖准备

输出：

1. 每个项目的高概率追问
2. 每题应答结构
3. 技术细节薄弱点
4. 建议补充的证据

### 3. 技术面模拟

输出：

1. 面试官问题
2. 用户回答点评
3. 继续追问
4. 更强版本的回答方向

### 4. 系统设计准备

输出：

1. 需求澄清问题
2. 核心架构思路
3. 可扩展性与风险点
4. 常见追问与应答方向

### 5. 面后复盘

输出：

1. 暴露的技术短板
2. 表达问题
3. 需要补的知识点或项目细节
4. 下一轮重点准备项

## Output Format

默认按以下结构输出：

1. 当前岗位目标
2. 主要风险点
3. 优先补强项
4. 可直接使用的回答或修改建议
5. 下一步准备动作

## Guardrails

- 不帮助用户伪造技术经历、指标或项目 ownership。
- 不把“技术词汇多”误判成“技术能力强”。
- 不用套路话术替代真实项目细节。
- 信息不足时，明确指出需要补充的上下文。

## Example

用户输入：

```text
我准备面 3-5 年后端岗位，帮我先看简历，再模拟一轮项目深挖和系统设计。
```

期望输出：

```text
先诊断简历中的技术风险点，再按项目深挖和系统设计分别做模拟与点评。
```
